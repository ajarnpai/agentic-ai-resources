Why NotebookLM + AI?

NotebookLM is Google's powerful AI tool that uses your own documents as the primary source of truth. Unlike general AI, it stays "grounded" in your research, notes, and data.

The Power of Connection: By linking NotebookLM to tools like Claude or Cursor via the Model Context Protocol (MCP), you give your favorite AI agents a "brain" full of your specific knowledge.

  • No more copying and pasting: Claude can read your entire project library instantly.
  • Fact-Checked Responses: Your AI uses your sources to answer, significantly reducing "hallucinations."
  • One-Stop Guide: This site provides everything you need to bridge that gap and build your custom technical knowledge base.

1. The Foundation

How to open your Terminal:
• 🪟 Windows: Press the `Windows` key, type PowerShell, and press Enter.
• 🍎 Mac: Press `Cmd + Space`, type Terminal, and press Enter.

🚫 Do Not Install Node.js

This tool is 100% Python-based. If prompted to install Node.js or Visual Studio Build Tools during any part of the setup, please skip or decline—they are not needed and will consume over 7GB of space.

Step 1: Python Engine

The "engine" that runs everything. Paste this into your Terminal:

Terminal
uv python install
Type these answers when asked:
• Add to PATH now? y
• Install CPython note? y
• View online help? n

✅ Check Ready: Run python --version

Step 2: uv Helper

The "app store" for Python. Paste this into your Terminal/PowerShell:

Installation (Windows/Mac)
uv-installer-command

✅ Check Ready: Run uv --version.
✨ Important: Restart your Terminal/PowerShell now!

Step 3: Install Tool

Install the actual NotebookLM connector in one hit.
⚠️ Required: Ensure you restarted your terminal after Step 2!

Terminal
uv tool install notebooklm-mcp-cli

✅ Check Ready: You should see "Installed 2 executables: nlm, notebooklm-mcp"

2. The Connection

Login to Google

Run this in your Terminal. It will open your web browser for a one-time Google login.

Terminal
nlm login

✅ Check Ready: You should see "Successfully authenticated!" and your email.

3. Setup Claude Desktop (Required)

⚠️ Windows Installer Warning

Install Claude Desktop using the direct .exe installer from claude.ai/download — NOT from the Microsoft Store. The Store version uses sandboxing that prevents it from reading the config file.

Step A: Locate or Create your Config File

Paste the path into your file explorer. If the Claude folder or json file doesn't exist, create them manually:
• 🪟 Windows: %AppData%\Claude\claude_desktop_config.json (Tip: Use Notepad, but ensure you save as .json and NOT .json.txt!)
• 🍎 Mac: ~/Library/Application Support/Claude/claude_desktop_config.json (Tip: Use TextEdit in Plain Text mode)

Step B: Add the JSON Config

If the file is NEW/EMPTY, paste this entire block:

Full Template (New File) 
{
  "mcpServers": {
    "notebooklm-mcp": {
      "command": "notebooklm-mcp"
    }
  }
}

⚠️ Reinstall Warning

After any reinstall of Claude Desktop on Windows, always re-check your config file. The installer can sometimes overwrite it with defaults, deleting your MCP entries. Re-add the mcpServers block if needed.

Step C: Verify the Config File

Run this in PowerShell to confirm your file has the correct content:

PowerShell
cat "$env:APPDATA\Claude\claude_desktop_config.json"

Step D: Restart Claude Desktop

✨ Important: Restart the Claude Desktop app completely after saving the file!

💡 Tip for Existing Configs (Windows): If you already have other MCP servers, run this PowerShell command to safely merge the NotebookLM entry:
$config = Get-Content "$env:APPDATA\Claude\claude_desktop_config.json" | ConvertFrom-Json; $config | Add-Member -Type NoteProperty -Name "mcpServers" -Value @{"notebooklm-mcp" = @{"command" = "notebooklm-mcp"}}; $config | ConvertTo-Json -Depth 10 | Set-Content "$env:APPDATA\Claude\claude_desktop_config.json"

6. Troubleshooting Guide

Issue 1: MCP Not Working (Config Correct)

⚠️ Windows Users: You might have the sandboxed Microsoft Store version of Claude. Check this by running:

PowerShell Check
ls "$env:LOCALAPPDATA\Packages\" | Where-Object {$_.Name -like "*Claude*"}

If you see Claude_pzs8sxrjxfjjc, uninstall it via Settings → Apps, then reinstall using the direct .exe from claude.ai/download.

Issue 2: Command Not Recognized

If you get a "not recognized" error when running notebooklm-mcp, the package isn't installed or not on PATH.

Terminal Fix
uv tool install notebooklm-mcp-cli

Then Close and Reopen your PowerShell window and try again.

Issue 3: Python/uv Not Working

Run these to verify installation and PATH:

Verify Core Tools
python --version; uv --version

⚠️ Critical: During Python installation, you MUST check "Add Python to PATH".

Issue 4: Auth Errors (Login Expired)

Your Google login session has expired. Re-authenticate:

Terminal Login
nlm login

This opens a browser for you to log in again. Sessions expire every 2–4 weeks.

Issue 5: Reinstalled Claude & MCP Stopped

The reinstaller overwrites your config. Re-add the MCP entry using this PowerShell merge command:

PowerShell Fix (Windows Only)
$config = Get-Content "$env:APPDATA\Claude\claude_desktop_config.json" | ConvertFrom-Json; $config | Add-Member -Type NoteProperty -Name "mcpServers" -Value @{"notebooklm-mcp" = @{"command" = "notebooklm-mcp"}}; $config | ConvertTo-Json -Depth 10 | Set-Content "$env:APPDATA\Claude\claude_desktop_config.json"

✨ Fully Restart Claude Desktop after running this!
